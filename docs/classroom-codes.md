# Klassen-Codes und Schüler-Registrierung

## Ziel

Diese Anleitung beschreibt den sicheren Ablauf für Lehrkräfte vor und während des Unterrichts.

## Vor dem Unterricht

1. Öffne `/teacher`.
2. Melde dich mit dem Lehrer*innen-Passwort an.
3. Trage unter `Neue Klasse anlegen` einen eindeutigen Klassennamen ein.
4. Klicke auf `Klasse erstellen`.
5. Prüfe den sofort angezeigten Klassen-Code im Bereich `Aktueller Klassen-Code`.
6. Klicke auf `Code kopieren`.
7. Lege fest:
   `Offene Version freigeben`, `SEB-Version freigeben`, `Aktive SEB-Lektion`, `Review-Lektion`.
8. Klicke auf `Einstellungen speichern`.

## Wenn ein Code ersetzt werden muss

1. Wähle die betroffene Klasse im Dashboard aus.
2. Klicke auf `Code neu erzeugen`.
3. Kopiere den neuen Code.
4. Teile ab diesem Moment nur noch den neuen Code mit der Lerngruppe.

## Schüler-Registrierung in der offenen Version

1. Schüler*in öffnet `/open`.
2. Klassen-Code exakt eingeben.
3. Eigenen Namen oder ein eindeutiges Kürzel eingeben.
4. Unterrichtspasswort eingeben.
5. Auf `Freischalten` klicken.

## Schüler-Registrierung in der SEB-Version

1. Schüler*in öffnet `/seb` im Safe Exam Browser.
2. Klassen-Code exakt eingeben.
3. Eigenen Namen oder ein eindeutiges Kürzel eingeben.
4. Auf `Starten` klicken.

## Kontrollliste direkt vor Unterrichtsbeginn

1. Prüfen, ob die richtige Klasse ausgewählt ist.
2. Prüfen, ob der sichtbare Klassen-Code aktuell ist.
3. Prüfen, ob die richtige SEB-Lektion gesetzt ist.
4. Prüfen, ob `Offene Version` oder `SEB-Version` passend aktiviert sind.
5. Einmal selbst mit einem Testnamen den Login auf `/open` oder `/seb` durchspielen.
6. Erst dann den Code an die Klasse weitergeben.

## Typische Fehlerquellen

- Falscher Klassen-Code: neuen Code kopieren und exakt weitergeben.
- Alte Klasse ausgewählt: vor dem Speichern immer den Klassennamen oben kontrollieren.
- Offene Version nicht freigegeben: `Offene Version freigeben` aktivieren und speichern.
- SEB gesperrt: Zugriff nur im Safe Exam Browser oder mit korrekt gesetztem SEB-Hash.
- Login schlägt in `/open` fehl: neben Code und Namen auch das Unterrichtspasswort prüfen.
